Индексы – это особые структуры данных, которые хранят малую часть данных коллекции в форме, удобной для поиска. Индекс хранит значение указанного поля или набора полей, причем поля упорядочены. 

Индексы обеспечивают эффективное выполнение запросов в MongoDB. Без индексов MongoDB выполняет сканирование коллекции, то есть просматривает каждый документ в коллекции, чтобы выявить те, которые соответствуют запросу. Если существует подходящий для запроса индекс, MongoDB может использовать индекс, чтобы ограничить количество документов, которые требуется просмотреть. MongoDB также может возвращать отсортированные результаты запроса, используя упорядочивание в индексе.

В MongoDB индекс определяется на уровне коллекции.

Чтобы создать индекс, следует использовать команду db.collection.createIndex():

db.collection.createIndex( <key and index type specification>, <options> )

Геопространственный индекс

Для обеспечения эффективного выполнения запросов по координатам, в MongoDB существует 2 специальных индекса:
2d индекс, который использует плоскостную геометрию
индекс 2dsphere, использующий сферическую геометрию.

Создание индекса 2dsphere

Для создания индекса 2dsphere следует использовать следующую команду:
db.collection.createIndex( { <location field> : "2dsphere" } )
где <location field> - это поле, значением которого является либо объект GeoJSON, либо пара координат.
